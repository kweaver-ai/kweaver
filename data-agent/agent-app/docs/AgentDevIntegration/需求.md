
背景：
我们有一个Agent统一开发平台，称作ADP，在这个平台上支持创建Agent，包括配置Agent，配置工具，配置技能，配置大模型，调整提示词等能力。配置完Agent后，可以发布Agent，将Agent发布到ADP的市场中，同时也可以指定Agent发布的类型，比如发布成API后，可以通过API的方式调用Agent进行问答。

现在，有外部的系统需要接入ADP，并且希望能够通过API的方式使用ADP创建的Agent来搭建一个对话的平台。我们需要提供一个完整的详细的Agent开发指南，用来指导如何进行系统级别的对接，以及如何应用Agent来搭建对话平台。

我们假定外部系统需要搭建的平台需要支持以下功能：
1. Agent的对话界面，侧边栏显示这个 Agent的会话列表。
2. 点击会话列表，可以进入会话，显示会话详情，也就是此会话中的多轮对话记录。
3. 输入框中输入问题，发送请求，得到答案。
4. 支持文件问答，文件上传临时区，在问答时，请求中包含临时区ID和文件ID，可以实现基于文件进行问答。
5. 支持对话恢复，也就是当浏览器刷新或者关闭后再打开，若对话尚未完成，也就是当前会话的状态为processing时，调用恢复接口，接着显示对话的答案。
6. 支持终止对话，点击终止按钮， 发送请求，终止后端服务中的对话。


补充信息：
1. 上述提到的功能均有相应的后端接口支持
2. 应该考虑账户鉴权体系不一致的问题，比如一个系统为AF，是和ADP为同一个公司内部的产品，使用的鉴权体系和账户体系是统一的，那么调用Agent API 时，传入的请求头的校验中，token 既可以使用普通用户的token，也可以使用应用账户的token。应用账户指的时，可以在ADP平台创建一个应用账户，生成应用账户token,token有效期很长，支持系统级别对接时进行API对接。 还有就是外部的系统，使用的账户鉴权体系与ADP完全不同，此时若想要使用ADP的Agent，需要先在ADP平台创建应用账户，然后使用应用账号的token调用API。
3. 考虑一个系统进行对接时，如何导入Agent。比如AF在对接ADP时，两个系统在部署后都是空白的，如果AF需要对接ADP。那么需要ADP部署后，通过接口导入内置的Agent，并发布为API ，或者在ADP界面创建并发布Agent，才能支持AF 的对接。
4. 对于Agent的API功能，如何进行对接
    - 4.1 这包含了Agent API 的接口文档定义，各参数的含义。
    - 4.2 请求参数发送示例
    - 4.3 会话管理，AgentAPI 支持会话自动创建机制，当用户发起第一次对话时，系统会自动创建会话并返回会话ID。后续可以使用这个会话ID继续对话。
    - 4.4 流式响应和非流式响应的不同
    - 4.5 响应结果的格式和各字段含义
    - 4.6 响应结果如何处理，若仅需要最终结果，可以取final_answer中的最终结果字段。 若需要展示对话过程各阶段调用，则使用middle_answer中的progress字段。
