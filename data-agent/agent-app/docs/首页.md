# Agent-App 项目文档

## 项目概述

Agent-App 是一个基于大模型的智能体应用平台，提供智能对话、任务执行和决策支持服务。项目采用 Go 语言开发，遵循领域驱动设计(DDD)架构，具有完整的微服务架构和可观测性支持。

## 核心特性

- 🤖 **智能对话**: 支持流式对话、多轮对话和会话管理
- 🔧 **Agent调试**: 提供Agent调试运行功能
- 📚 **文档召回**: 支持文档检索和引用
- 🔄 **会话恢复**: 支持中断对话的恢复和继续
- 🎯 **权限控制**: 完整的用户和应用权限校验
- 📊 **可观测性**: 完整的监控、日志和追踪体系
- 🚀 **高性能**: 使用高性能JSON序列化库Sonic

## 技术栈

- **语言**: Go 1.23.7
- **Web框架**: Gin
- **数据库**: MySQL (通过proton-rds-sdk-go)
- **消息队列**: Kafka
- **配置管理**: Viper
- **序列化**: Sonic (高性能JSON序列化)
- **可观测性**: OpenTelemetry
- **容器化**: Docker + Kubernetes (Helm)

## 主要模块

- **智能体服务 (agentsvc)**: 核心对话逻辑和会话管理
- **会话管理**: 多轮对话、会话列表、已读状态
- **API接口**: RESTful API接口
- **持久化层**: 数据库访问和会话存储
- **外部服务集成**: Agent Factory、Agent Executor等

## 核心能力

### 智能对话能力
- **普通对话**: 基础的问答式对话，支持单次查询
- **多轮对话**: 支持基于历史上下文的连续对话
- **API对话**: 将Agent能力封装为API服务
- **人工干预对话**: 支持人工介入对话流程
- **调试功能**: 完整的调试工具和监控能力

### 会话管理能力
- **会话生命周期管理**: 创建、查询、更新、删除
- **消息管理**: 完整的消息生命周期管理
- **临时区域管理**: 管理会话中的临时数据和中间状态
- **上下文维护**: 自动维护对话上下文和历史记录

### 流式处理能力
- **增量流式传输**: 只传输变化的数据，减少网络传输量
- **全量流式传输**: 每次传输完整的响应数据
- **实时响应机制**: 支持Server-Sent Events (SSE)
- **连接管理**: 支持SSE长连接和优雅关闭

### 性能优化特性
- **缓存策略**: 会话缓存、进度缓存、配置缓存
- **数据库优化**: 连接池、索引优化、批量操作
- **流式处理优化**: 增量传输、批量处理、连接复用

## 部署方式

- **生产环境**: Kubernetes + Helm
