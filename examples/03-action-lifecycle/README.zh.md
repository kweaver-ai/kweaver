# 03 · 行动生命周期 — 自我进化的知识网络

> 一个能自主监控供应链并采取行动的知识网络。

## 这个示例展示什么

知识网络不是静态的查询层。定义**行动类型**和**调度计划**之后，它开始自主运转：

- **发现正确的实体** — 基于关系感知的条件（例如："关联了高风险供应商的采购单"）
- **触发跟进行动** — 调用你的业务系统
- **自动留存记录** — 完整的操作审计在 `action-log` 中

不再需要每天早上手动跑脚本。知识网络自己行动，你只需查看日志。

## 前置条件

- `kweaver` CLI ≥ 0.6.3，且已登录（`kweaver auth whoami`）
- MySQL 可从 kweaver 平台访问
- 本机安装 Python 3 和 `curl`

## 快速开始

```bash
cp env.sample .env
# 填写 .env 中的数据库信息
./run.sh
```

## 流程说明

| 步骤 | 发生了什么 |
|------|-----------|
| 0 | 注入 MySQL 数据：供应商 + 采购单（4 条关联高风险供应商） |
| 1–2 | 连接数据源，构建知识网络 |
| 3–5 | 注册行动工具后端 |
| 6 | 定义行动类型：*"找到高风险供应商的采购单，触发跟进"* |
| 7 | 查询确认：通过关系上下文发现受影响的采购单 |
| 8–9 | 设定调度：每天早上 08:00 自动运行 |
| 10 | 手动触发一次，立即查看结果 |
| 11 | 审计日志：知识网络自主行动的完整记录 |

## 关于执行状态

Demo 工具后端不做真实的数据写回（写回是你的业务系统的职责）。执行结果在
工具层面可能显示 `failed`，但执行记录和审计日志仍然正常写入。生产环境中，
将工具绑定替换为你的 ERP 或业务 API 接口即可。

## 资源清理

脚本退出时（无论成功或失败）自动清理所有创建的资源。
